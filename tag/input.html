<input> タグ

HTMLの <input> 要素は、Form control を表します。 フォーム入力の中心的な要素で、用途に応じて type を選びます。 この要素は空要素で終了タグを持ちません。 ラベルと関連付けると操作しやすくなります。 関連タグ: <label>、<form>。

属性

固有属性（クリックで開く）
accept

何を表すか: 受け付けるファイル種別を指定します。例: image/* で画像だけ、.pdf でPDFだけに制限します。

書き方: accept="値"

選択肢:

  • カンマ区切り: 複数キーワードを指定

例: <input accept="例">

alpha

何を表すか: 指定の有無で有効/無効を切り替えます。指定するとON、外すとOFFになります。

書き方: 属性名だけ書く（例: alpha）

選択肢:

  • 付与: 有効
  • 省略: 無効

例: <input alpha>

alt

何を表すか: 画像などが表示できないときに代わりに示す説明文です。読み上げにも使われます。

ポイント: 画像が表示できないときの代わりの説明です。内容が伝わる短い文章にします。

書き方: alt="値"

例: <input alt="例">

autocomplete

何を表すか: 入力補完の方針を指定します。例: email や ame を指定して、ブラウザに入力の意味を伝えます。

書き方: autocomplete="値"

選択肢:

  • on: 補完を許可
  • off: 補完を無効化
  • 用途別トークン: 例: name, email, username, current-password など

例: <input autocomplete="例">

checked

何を表すか: チェックボックスやラジオの初期選択状態を指定します。

ポイント: 初期状態で選択されていることを示します。

書き方: 属性名だけ書く（例: checked）

選択肢:

  • 付与: 有効
  • 省略: 無効

例: <input checked>

colorspace

何を表すか: 使う色空間を指定します（対応環境限定）。

書き方: colorspace="値"

例: <input colorspace="例">

dirname

何を表すか: 入力方向を送信データに含めます。指定すると name.dir の形で方向情報が送信されます。

書き方: dirname="値"

例: <input dirname="例">

disabled

何を表すか: 操作できない状態にします。送信対象にもならない点に注意します。

ポイント: 操作できない状態にします。見た目も変わることが多いです。

書き方: 属性名だけ書く（例: disabled）

選択肢:

  • 付与: 有効
  • 省略: 無効

例: <input disabled>

form

何を表すか: この要素がどのformに属するかを指定します。フォームの外に置いた部品を、特定のフォームに結び付けたいときに使います。

書き方: form="値"

例: <input form="例">

formaction

何を表すか: URLを指定します。例: 読み込むファイルや移動先のURLを入れます。

書き方: formaction="値"

例: <input formaction="例">

formenctype

何を表すか: フォーム送信時のデータのエンコード方式を指定します。例: application/x-www-form-urlencoded / multipart/form-data / text/plain。

書き方: formenctype="値"

例: <input formenctype="例">

formmethod

何を表すか: 送信時のHTTPメソッドを指定します。例: get や post。

書き方: formmethod="値"

例: <input formmethod="例">

formnovalidate

何を表すか: 指定の有無で有効/無効を切り替えます。指定するとON、外すとOFFになります。

書き方: 属性名だけ書く（例: formnovalidate）

選択肢:

  • 付与: 有効
  • 省略: 無効

例: <input formnovalidate>

formtarget

何を表すか: 送信結果の表示先を指定します。例: _self（同じタブ）/_blank（新しいタブ）。

書き方: formtarget="値"

例: <input formtarget="例">

height

何を表すか: 表示高さを指定します。例: 200 のように数値で枠の高さを決めます。

書き方: height="値"

例: <input height="例">

list

何を表すか: 参照するdatalistのidを指定し、入力候補リストと結び付けます。

書き方: list="値"

例: <input list="例">

max

何を表すか: 数値入力の最大値を指定します。これより大きい値は受け付けません。

ポイント: 数値の上限を決めます。min とセットで使うことが多いです。

書き方: max="値"

例: <input max="例">

maxlength

何を表すか: 数値を指定します。例: 3 と書けば3つ分・3行分など、数量や範囲を数で指定します。

書き方: maxlength="値"

例: <input maxlength="例">

min

何を表すか: 数値入力の最小値を指定します。これより小さい値は受け付けません。

ポイント: 数値の下限を決めます。入力できる範囲がはっきりします。

書き方: min="値"

例: <input min="例">

minlength

何を表すか: 数値を指定します。例: 3 と書けば3つ分・3行分など、数量や範囲を数で指定します。

書き方: minlength="値"

例: <input minlength="例">

multiple

何を表すか: 複数選択や複数入力を許可します。複数の項目を同時に選ばせたいときに使います。

書き方: 属性名だけ書く（例: multiple）

選択肢:

  • 付与: 有効
  • 省略: 無効

例: <input multiple>

name

何を表すか: フォーム送信時の「項目名」になる属性です。サーバー側ではこの名前で値を受け取ります。

ポイント: 送信時のキーになる名前です。フォームでは必須になることが多いです。

書き方: name="値"

例: <input name="例">

pattern

何を表すか: 入力できる文字列の規則を正規表現で指定します。条件に合わないと送信できません。

ポイント: 入力のルールを正規表現で指定します。初心者は簡単なパターンから始めると安心です。

書き方: pattern="値"

例: <input pattern="例">

placeholder

何を表すか: 入力欄に薄いヒント文字を表示します。入力すると消えます。

書き方: placeholder="値"

例: <input placeholder="例">

popovertarget

何を表すか: 開閉するpopover要素のidを指定します。

書き方: popovertarget="値"

例: <input popovertarget="例">

popovertargetaction

何を表すか: popoverの動作を指定します。例: show/hide/toggle。

書き方: popovertargetaction="値"

例: <input popovertargetaction="例">

readonly

何を表すか: 読み取り専用にします。表示はされますが編集できません。

ポイント: 表示はするが編集できない状態にします。

書き方: 属性名だけ書く（例: readonly）

選択肢:

  • 付与: 有効
  • 省略: 無効

例: <input readonly>

required

何を表すか: 必須入力であることを示します。空欄だと送信できなくなります。

ポイント: 必須入力にする属性です。

書き方: 属性名だけ書く（例: required）

選択肢:

  • 付与: 有効
  • 省略: 無効

例: <input required>

size

何を表すか: 数値を指定します。例: 3 と書けば3つ分・3行分など、数量や範囲を数で指定します。

書き方: size="値"

例: <input size="例">

src

何を表すか: 画像ボタンで使う画像のURLを指定します。type='image' のときに利用します。

ポイント: type='image' のときに表示する画像ファイルの場所を指定します。

書き方: src="値"

例: <input src="例">

step

何を表すか: 数値入力の刻み幅を指定します。スピンボタンの増減にも影響します。

ポイント: 数値の増減の刻み幅を決めます。

書き方: step="値"

例: <input step="例">

type

何を表すか: 要素の種類やデータの形式を指定します。指定するとブラウザが正しく解釈しやすくなります。

ポイント: 要素の種類や形式を決めます。用途が複数ある要素では特に重要です。

書き方: type="値"

例: <input type="例">

value

何を表すか: フォームで送信される値や初期値を指定します。実際に送るデータ内容に直結します。

ポイント: 送信される値や初期値を決めます。何が送られるかを意識して書きます。

書き方: value="値"

例: <input value="例">

width

何を表すか: 表示幅を指定します。例: 300 のように数値で枠の幅を決めます。

書き方: width="値"

例: <input width="例">

コード例

<label>名前 <input type="text" name="name"></label>
