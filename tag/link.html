<link> タグ

HTMLの <link> 要素は、Link metadata を表します。 外部のCSSなど、関連リソースを読み込むために使います。 この要素は空要素で終了タグを持ちません。 主に <head> 内で使います。 関連タグ: <style>、<meta>。

外部CSSを読み込む代表例は rel="stylesheet" と href です。例: <link rel="stylesheet" href="style.css">。

属性

固有属性（クリックで開く）
href

何を表すか: リンク先を指定する属性です。クリックするとこのURLへ移動します。相対パスも使えます。

ポイント: リンク先を決める一番大切な属性です。まずは有効なURLか相対パスを書くことを意識します。

書き方: href="値"

例: <link href="例">

crossorigin

何を表すか: 外部スクリプト取得時のCORS設定を指定します。SRI（integrity）と組み合わせるときによく使います。

書き方: crossorigin="値"

選択肢:

  • anonymous: 資格情報（Cookie等）を送らない
  • use-credentials: 資格情報を送る

例: <link crossorigin="例">

rel

何を表すか: リンク先との関係を指定します。スタイルシート・外部参照・nofollow など用途を伝えます。

ポイント: リンクの関係を示します。外部リンクやスタイル指定などで使います。

書き方: rel="値"

選択肢:

  • スペース区切り: 複数キーワードを指定

例: <link rel="例">

as

何を表すか: 読み込むリソースの種類を指定します。例: style/script/font/image など、正しい種類を書くと最適化が働きます。

書き方: as="値"

例: <link as="例">

media

何を表すか: 適用するメディア条件を指定します。例: (min-width: 600px) のように条件を書き、条件に合うときだけ有効にします。

書き方: media="値"

例: <link media="例">

hreflang

何を表すか: リンク先の言語を指定します。例: 英語ページなら en、日本語なら ja を指定します。

書き方: hreflang="値"

例: <link hreflang="例">

type

何を表すか: 要素の種類やデータの形式を指定します。指定するとブラウザが正しく解釈しやすくなります。

ポイント: 要素の種類や形式を決めます。用途が複数ある要素では特に重要です。

書き方: type="値"

例: <link type="例">

sizes

何を表すか: レスポンシブ画像の表示幅条件を指定します。例: 100vw や (min-width: 600px) 50vw のように条件を書きます。

書き方: sizes="値"

選択肢:

  • スペース区切り: 複数キーワードを指定

例: <link sizes="例">

imagesrcset

何を表すか: レスポンシブ画像の候補URLを指定します。srcset と同じ形式で複数の候補を並べます。

書き方: imagesrcset="値"

選択肢:

  • カンマ区切り: 複数キーワードを指定

例: <link imagesrcset="例">

imagesizes

何を表すか: 画像の表示幅条件を指定します。sizes と同じ形式で書きます。

書き方: imagesizes="値"

例: <link imagesizes="例">

referrerpolicy

何を表すか: 外部スクリプト取得時に送るリファラ（参照元情報）の方針を指定します。

書き方: referrerpolicy="値"

選択肢:

  • no-referrer: リファラを送らない
  • no-referrer-when-downgrade: HTTPS→HTTPでは送らない
  • origin: オリジンのみ送る
  • origin-when-cross-origin: 同一オリジンは完全、他はオリジンのみ
  • same-origin: 同一オリジンのみ送る
  • strict-origin: オリジンのみ、ダウングレード時は送らない
  • strict-origin-when-cross-origin: 同一は完全、他はオリジンのみ、ダウングレード時は送らない
  • unsafe-url: 完全なURLを常に送る

例: <link referrerpolicy="例">

integrity

何を表すか: 外部ファイルのハッシュ値を指定して改ざん検知を行います。CDN利用時によく使います。

書き方: integrity="値"

例: <link integrity="例">

blocking

何を表すか: 読み込み時にブロックする対象を指定します。レンダリングを止める用途で使います。

書き方: blocking="値"

選択肢:

  • スペース区切り: 複数キーワードを指定

例: <link blocking="例">

color

何を表すか: アイコンなどに使う色を指定します。例: #0f172a のような色コードを入れます。

書き方: color="値"

例: <link color="例">

disabled

何を表すか: 操作できない状態にします。送信対象にもならない点に注意します。

ポイント: 操作できない状態にします。見た目も変わることが多いです。

書き方: 属性名だけ書く（例: disabled）

選択肢:

  • 付与: 有効
  • 省略: 無効

例: <link disabled>

fetchpriority

何を表すか: リソース取得の優先度を指定します。読み込み順序に影響します。

書き方: fetchpriority="値"

例: <link fetchpriority="例">

コード例

<!doctype html>
<html lang="ja">
<head>
  <link rel="stylesheet" href="style.css">
</head>
<body>...</body>
</html>
