<script> タグ

HTMLの <script> 要素は、Embedded script を表します。 JavaScriptを読み込んだり実行したりするための要素です。 外部ファイルを使う場合は src を指定します。 読み込み位置で動作が変わるため配置に注意します。 関連タグ: <noscript>、<head>。

外部ファイルを読み込む場合は src を指定します。例: <script src="app.js"></script>。

属性

固有属性（クリックで開く）
src

何を表すか: 読み込むJavaScriptファイルの場所を指定します。外部ファイルに書いた処理を読み込ませるための指定です。

ポイント: JavaScriptファイルの相対パスやURLを指定します。例: js/app.js のように書くと構成が分かりやすいです。

書き方: src="値"

例: <script src="例"></script>

type

何を表すか: スクリプトの種類（通常のJavaScriptか、ES Modulesか）を指定します。

ポイント: モジュールとして読み込む場合は module を指定します。通常のJavaScriptは省略しても動きます。

書き方: type="値"

選択肢:

  • text/javascript: 通常のJavaScript（多くの場合は省略可）
  • module: ES Modulesとして読み込む

例: <script type="例">...</script>

nomodule

何を表すか: モジュールに対応したブラウザでは読み込ませないための属性です。古いブラウザ向けのフォールバック用に使います。

書き方: 属性名だけ書く（例: nomodule）

選択肢:

  • 付与: 有効
  • 省略: 無効

例: <script nomodule>...</script>

async

何を表すか: 外部スクリプトを並列で読み込み、読み込み完了後すぐに実行します。実行順は保証されません。

書き方: 属性名だけ書く（例: async）

選択肢:

  • 付与: 有効
  • 省略: 無効

例: <script async>...</script>

defer

何を表すか: HTMLの解析が終わった後に実行されます。複数指定した場合は記述順に実行されます。

書き方: 属性名だけ書く（例: defer）

選択肢:

  • 付与: 有効
  • 省略: 無効

例: <script defer>...</script>

crossorigin

何を表すか: 外部スクリプト取得時のCORS設定を指定します。SRI（integrity）と組み合わせるときによく使います。

書き方: crossorigin="値"

選択肢:

  • anonymous: 資格情報（Cookie等）を送らない
  • use-credentials: 資格情報を送る

例: <script crossorigin="例">...</script>

integrity

何を表すか: 外部ファイルのハッシュ値を指定して改ざん検知を行います。CDN利用時によく使います。

書き方: integrity="値"

例: <script integrity="例">...</script>

referrerpolicy

何を表すか: 外部スクリプト取得時に送るリファラ（参照元情報）の方針を指定します。

書き方: referrerpolicy="値"

選択肢:

  • no-referrer: リファラを送らない
  • no-referrer-when-downgrade: HTTPS→HTTPでは送らない
  • origin: オリジンのみ送る
  • origin-when-cross-origin: 同一オリジンは完全、他はオリジンのみ
  • same-origin: 同一オリジンのみ送る
  • strict-origin: オリジンのみ、ダウングレード時は送らない
  • strict-origin-when-cross-origin: 同一は完全、他はオリジンのみ、ダウングレード時は送らない
  • unsafe-url: 完全なURLを常に送る

例: <script referrerpolicy="例">...</script>

blocking

何を表すか: 読み込み中に何をブロックするかを指定します。指定すると描画を止めるため、基本は必要なときだけ使います。

書き方: blocking="値"

選択肢:

  • render: ページの描画を一時停止する

例: <script blocking="例">...</script>

fetchpriority

何を表すか: 取得の優先度を指定します。読み込みの順序やタイミングに影響します。

書き方: fetchpriority="値"

選択肢:

  • auto: ブラウザに任せる（既定）
  • high: 優先度を上げる
  • low: 優先度を下げる

例: <script fetchpriority="例">...</script>

コード例

<script>console.log("hello");</script>
