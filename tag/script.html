<script> タグ

HTMLの <script> 要素は、Embedded script を表します。 JavaScriptを読み込んだり実行したりするための要素です。 外部ファイルを使う場合は src を指定します。 読み込み位置で動作が変わるため配置に注意します。 関連タグ: <noscript>、<head>。

外部ファイルを読み込む場合は src を指定します。例: <script src="app.js"></script>。

属性

固有属性（クリックで開く）
src
意味: リソースのURLを指定します。
指定方法: src="値"
type
意味: 要素の種類を指定します。
指定方法: type="値"
nomodule
意味: 指定の有無で有効/無効を切り替えます。
指定方法: 属性名だけ書く（例: nomodule）
選択肢:
  • 付与: 有効
  • 省略: 無効
async
意味: 指定の有無で有効/無効を切り替えます。
指定方法: 属性名だけ書く（例: async）
選択肢:
  • 付与: 有効
  • 省略: 無効
defer
意味: 指定の有無で有効/無効を切り替えます。
指定方法: 属性名だけ書く（例: defer）
選択肢:
  • 付与: 有効
  • 省略: 無効
crossorigin
意味: CORSの送信方式を指定します。
指定方法: crossorigin="値"
選択肢:
  • anonymous: 資格情報（Cookie等）を送らない
  • use-credentials: 資格情報を送る
integrity
意味: 任意のテキストを指定します。
指定方法: integrity="値"
referrerpolicy
意味: リファラ送信方針を指定します。
指定方法: referrerpolicy="値"
選択肢:
  • no-referrer: リファラを送らない
  • no-referrer-when-downgrade: HTTPS→HTTPでは送らない
  • origin: オリジンのみ送る
  • origin-when-cross-origin: 同一オリジンは完全、他はオリジンのみ
  • same-origin: 同一オリジンのみ送る
  • strict-origin: オリジンのみ、ダウングレード時は送らない
  • strict-origin-when-cross-origin: 同一は完全、他はオリジンのみ、ダウングレード時は送らない
  • unsafe-url: 完全なURLを常に送る
blocking
意味: キーワード（複数指定可）を指定します。
指定方法: blocking="値"
選択肢:
  • スペース区切り: 複数キーワードを指定
fetchpriority
意味: 用途に応じた値を指定します。
指定方法: fetchpriority="値"

コード例

<script>console.log("hello");</script>
